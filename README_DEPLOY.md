Deploy rápido (gratis) - Express backend + Supabase (Postgres)\n\nOpções recomendadas:\n- Backend: Railway (grátis) ou Render (grátis tier)\n- Frontend: Cloudflare Pages (integra com Cloudflare)\n- Banco: Supabase (já em uso)\n\nPassos (Railway/Render):\n1. Faça push do repositório para GitHub.\n2. Crie um projeto no Railway ou serviço web no Render e conecte o repositório.\n3. Configure variáveis de ambiente:\n   - `DATABASE_URL` = string de conexão Postgres do Supabase (use a connection string com password).\n   - (Opcional) `PORT` = 3000 (a plataforma fornece automaticamente).\n4. Defina o comando de start (Render/Railway usam `npm start` por padrão). O `Procfile` contém `web: node server.js` caso a plataforma use Procfile.\n5. Deploy e verifique a rota `/api/health` para testar.\n\nDicas para usar Cloudflare Pages (frontend):\n- Crie um projeto no Cloudflare Pages apontando para a pasta com o frontend (separar build).\n- Configure variáveis do ambiente no Pages (se necessário) para apontar ao backend.\n\nMigrar para Workers (opcional):\n- Se quiser reduzir custos e usar edge functions, remova dependência de `pg` no worker e use `@supabase/supabase-js` com a service key para acessar o banco via HTTP.\n\nTestes locais:\n```bash\n# Instale dependências\nnpm install\n# Rodar backend\nnpm run backend\n# Executar frontend (em outra aba)\nnpm run dev\n```\n
